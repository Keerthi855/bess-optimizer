HOW THE CALCULATION WORKS

Step 1 — Annualize Battery Cost

CRF = r·(1+r)ⁿ / ((1+r)ⁿ−1)
// Capital Recovery Factor converts upfront cost → annual
CAPEX_annual = battery_cost × CRF × E_kWh
→A 10-year, 8% discount rate gives CRF ≈ 0.149 — meaning you "pay" 14.9% of the capital cost each year.

Step 2 — Objective Function (what Gurobi minimizes)

minimize:
  CAPEX_ann (€/yr)
 + Σ import_price × imp[t] (€/yr)
 − Σ export_price[t] × exp[t] (€/yr)
// When export_price[t] < 0:
// −(negative × exp) = +cost → optimizer avoids exporting!

Step 3 — Power Balance (every hour)

solar[t] + discharge[t] + import[t]
 = load[t] + ch_solar[t] + ch_grid[t] + export[t]
→Every kWh must be accounted for. No free energy, no lost energy.

Step 4 — Negative Price Logic

When export_price[t] < 0:
  Exporting costs money (penalized in objective)
  → optimizer sets exp[t] ≈ 0
  → surplus solar goes to load first
  → then to battery (ch_solar[t])
  → only unavoidable surplus goes to grid
// No hard constraint needed — economics drive this

Step 5 — Battery Power Cap

P_cap ≤ c_rate × E_cap
// No solar inverter cap — Gurobi finds optimal P freely
// Only constraint: P ≤ c_rate × E (e.g. 0.5C → 2-hour battery)

Step 6 — SOC Dynamics

SOC[t] = SOC[t−1]
  + (ch_sol[t−1] + ch_grd[t−1]) × η_charge
  − discharge[t−1] / η_discharge

E × soc_min ≤ SOC[t] ≤ E × soc_max
→Losses are modelled in both directions. Cycling within min/max SOC protects battery life.

Step 7 — Optimal Size Output

Gurobi finds E_kWh and P_kW that minimize
total annual cost over all 8760 hours.
// Smaller battery → lower CAPEX but higher grid cost
// Larger battery → higher CAPEX but lower grid cost
Optimal point = where ΔCAPEX = ΔGrid savings
